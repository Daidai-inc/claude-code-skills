# Skill: computer-use — ブラウザ・GUI操作の自動化

MCPもAPIも使えない場面で、ClaudeがMacの画面を直接見て操作する。
ログイン済みのブラウザセッションをそのまま使えるため、専用の接続コード不要。

## 発動優先順位（必ず上から順に検討）

| 優先度 | 手段 | 判断基準 |
|--------|------|---------|
| 1 | MCP | 対象サービスのMCPが存在する（kintone, Google Drive, GitHub等） |
| 2 | API / CLI | REST API・CLIが使える（Vercel CLI, GitHub CLI, npm等） |
| 3 | **Computer Use** | MCP/APIがなく、ブラウザGUI操作が必要な場合に発動 |

## トリガー（このスキルを発動する条件）

以下の条件を**全て**満たす場合に発動を提案する:
- ユーザーが手動でやっていたブラウザ操作
- MCPもAPIも存在しない or 使えない
- 繰り返し発生する or 今すぐ代わりにやってほしい

トリガーキーワード例:
- 「ログインして〜を確認して」
- 「〜のフォームを入力して」
- 「〜のサイトで〜を申し込んで」
- 「MCPないけど自動化したい」

## 実行方法

### Dispatch経由（推奨・非同期）
iPhoneから指示して結果を受け取る。Macが起動している必要あり。
```
claude.ai/dispatch → タスクを自然言語で指示
例:「ainagoc.netのTOPICSページを開いて、IT・ボランティア管理に関する新着案件を探して報告して」
```

### 対話セッション中（直接）
```
「Computer Useで〜をやって」と指示する
→ Claudeが画面を開いて操作を開始
```

## コスト管理

- トークン消費: MCP/APIの5〜20倍
- 向いている用途: 月1〜週1回の繰り返し作業、フォーム入力、ページ確認
- 向かない用途: 毎日実行するルーティン（MCP/APIに移行すべき）、大量データ処理

## 実行後の記録

Computer Useで完了した作業は以下に記録する:
- 画面操作の結果をテキストでまとめてユーザーに報告
- 繰り返し使う作業はtasks.mdに定期タスクとして登録
- 将来MCPが使えるようになったらMCPに移行する
