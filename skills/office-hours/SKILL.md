# Skill: office-hours — アイデア検証・クイック相談

新しいアイデアや仮説を素早くブレストし、「進める価値があるか」「穴はどこか」を短時間で判断する。深い分析より速さと率直さを優先する。

## トリガー
- 「ちょっと相談したい」「アイデアがあるんだけど」「思いついたんだけど」
- 「これどう思う？」（軽い確認文脈。重要な意思決定は /plan-ceo-review へ）
- 「穴あるかな」「どこがネックになりそう？」
- `/office-hours`

## 実行ステップ

1. **アイデアを1文で言い換える**（理解確認 + 本質の明確化）

2. **即興評価（3点）**
   - 良い点: 一番刺さる部分を1〜2点
   - 穴: 最大の弱点・見落としを1点
   - 前提確認: 成立に必要な仮定（間違ってると全部崩れる）を1点

3. **判定**
   ```
   →  [続けて深掘り / /plan-ceo-reviewで正式検討 / いったん寝かせる / やめとこう]
   ```

4. **もし続けるなら**: 次の1アクションを1行で提示

## 使い分けの目安

| シチュエーション | スキル |
|----------------|--------|
| 思いつきのブレスト | /office-hours |
| 正式な意思決定（お金・契約・方針変更） | /plan-ceo-review |
| 実装・技術選定の判断 | /plan（planスキル） |

## 出力フォーマット

全体で10行以内。箇条書き中心で。

```
一言で言うと: [アイデアの本質]

良い点: ...
穴: ...
前提: ...

→ [続けて / 正式検討へ / 寝かせる / やめとこう]
次の一手: ...
```

## Gotchas（よくある失敗パターン）

- **分析が深くなりすぎる**: office-hoursは"まず5分で考える"フェーズ。深掘りはplan-ceo-reviewの仕事
- **褒めすぎて穴を言わない**: 率直な一番の弱点を必ず1点は出す
- **判定を出さない**: 「面白いですね」で終わらず、続ける/やめるを明示する
- **複数アイデアを同時に評価する**: 1回1アイデアに絞る。まとめて評価すると穴が見えなくなる
