# Skill: careful — 本番操作の安全モード

破壊的コマンドや本番に影響する操作の前に警告を出し、確認を求める。AXISの本番データ・Azure設定・DB操作での誤操作を防ぐ。

## トリガー
- 「慎重にやって」「本番作業」「本番DB」「production」
- 「データ削除」「マイグレーション実行」「ロールバック」
- 「force push」「reset --hard」「DROP TABLE」
- `/careful`

## 対象となる操作（警告を出す）

### 赤（確認必須 — 実行前に必ず止まる）
- DB操作: `DROP`, `DELETE FROM`, `UPDATE ... WHERE`なし, Prismaマイグレーション実行
- Git: `reset --hard`, `push --force`, `branch -D`, `checkout .`
- ファイル: `rm -rf`, 本番設定ファイルの上書き
- Azure: App Serviceの停止・設定変更・スロットスワップ
- 環境変数: 本番の`.env`直接編集

### 黄（情報提供 — 代替案を提示してから実行）
- `git stash`（未コミット作業が消える可能性）
- `npm install --save`（package-lock.json変更）
- Prismaスキーマ変更（マイグレーション生成が必要）

## 実行フロー

1. **操作を検出**
2. **影響範囲を明示**:
   ```
   ⚠️  [操作名]
   影響: [何が変わるか・消えるか]
   取り消し可否: [可能 / 困難 / 不可能]
   ```
3. **代替案を提示**（あれば）
4. **確認を求める**: 「実行しますか？ [yes/no]」

## 本番作業チェックリスト

DB変更前:
- [ ] バックアップ確認（Supabase/Azureのスナップショット）
- [ ] マイグレーションのロールバックSQL準備
- [ ] メンテナンスウィンドウの通知

Azure設定変更前:
- [ ] スロットスワップの影響（セッション・接続プール）
- [ ] 環境変数変更後の再起動確認
- [ ] ヘルスチェックエンドポイントの準備

Git操作前:
- [ ] 作業ブランチのバックアップ（`git branch backup/...`）
- [ ] チームへの通知（退職後・チーム開発時）

## /guard モード（combined）

`/guard` = `/careful` + ディレクトリ制限（デバッグ中に他ファイルを誤変更しない）

```
/guard src/app/[feature]/  # このディレクトリのみ編集可
```

## Gotchas（よくある失敗パターン）

- **警告を出すだけで止まらない**: 赤操作は実行前に必ず止まる。「多分大丈夫」で通さない
- **ローカルでも本番データを触る場合がある**: .env.localでも本番DB URLが入っているケースに注意
- **Prismaマイグレーションの取り消しは困難**: `migrate dev`は開発用。本番は`migrate deploy`で慎重に
- **force pushは当該ブランチを指定確認**: `--force-with-lease`を推奨。main/masterへのforce pushは拒否
