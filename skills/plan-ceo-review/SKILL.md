# Skill: plan-ceo-review — CEO視点での意思決定レビュー

戦略的な意思決定・提案・計画をCEO視点で深掘り分析し、承認/修正/棄却の判断と根拠を提供する。

## トリガー
- 「CEOレビューして」「経営判断を仰ぎたい」「意思決定支援」
- 「この提案どう思う？」「進めていい？」「判断して」（重要な決定文脈で）
- `/plan-ceo-review`

## 想定インプット例
- JTBへの提案内容・契約条件
- 新機能・新事業の採否判断
- 採用・外注・パートナーシップの意思決定
- スケジュール変更・タイムライン修正
- コスト・投資判断

## 実行ステップ

1. **コンテキスト確認**
   - 判断対象を1文で要約する
   - 決定期限・制約（公務員制約、資金、タイムライン）を確認
   - 判断に必要な情報が不足していれば質問を1〜2個に絞る（聞きすぎない）

2. **5視点での分析**（各視点を2〜3行で簡潔に）

   | 視点 | 問い |
   |------|------|
   | 財務 | 収支インパクト・キャッシュフロー・ROI |
   | リスク | 最悪シナリオ・回復可能性・法的リスク |
   | 機会 | やらないことのコスト・競合との差 |
   | タイミング | 今やるべき理由・待つと何を失うか |
   | リソース | 一人開発体制での実現可能性・優先度の押しのけ |

3. **判断**
   ```
   判断: [承認 / 条件付き承認 / 要修正 / 棄却]
   理由: （1〜2文）
   条件/修正点: （あれば箇条書きで最大3点）
   ```

4. **ネクストアクション**
   - 今すぐやること（24時間以内）を1〜2行で明示
   - キューに積むべきなら `bash scripts/queue_task.sh -p 2 -n "アクション名" "プロンプト"` コマンドを提示

## AXISプロジェクト固有の判断基準

常にこれらの文脈を加味する:
- 在職中の制約: 法人設立・営利活動・GビズID申請は5月以降
- アジア競技大会タイムライン: 9月19日〜10月4日が絶対期限
- JTBパートナーシップ: LOI/MOU締結が5〜6月目標
- 一人開発体制: リソース分散は致命的。集中が原則
- 資金: 100〜500万円の資本金範囲内で判断

## 出力フォーマット

簡潔を優先。全体で15〜20行以内を目安にする。

```
## CEOレビュー: [判断対象の一言要約]

**状況**: [背景を1〜2文]

**分析**
- 財務: ...
- リスク: ...
- 機会: ...
- タイミング: ...
- リソース: ...

**判断**: 承認 / 条件付き承認 / 要修正 / 棄却
**理由**: ...
**条件**: （あれば）

**今すぐやること**: ...
```

## Gotchas（よくある失敗パターン）

- **分析しすぎて判断を出さない**: 全視点を埋めることより「承認/棄却」の結論を出すことが目的。情報不足でも確率的判断を出す
- **現在の制約を無視した理想論**: 公務員制約・一人体制・9月期限を常に前提に入れる
- **ネクストアクションが曖昧**: 「検討する」ではなく「今週中に〇〇に連絡する」のように具体的にする
- **全項目を長文で書く**: 各視点は2〜3行が上限。CEOは短い要約で判断する
- **JTBとAXIS本体の混在**: JTB向け提案とAXIS機能開発は別のリソースプールとして扱う
